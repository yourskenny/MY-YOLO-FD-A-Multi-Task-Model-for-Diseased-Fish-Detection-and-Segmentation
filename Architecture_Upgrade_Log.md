# YOLO-FD 架构升级溯源文档 (Plan A + B)

## 1. 改动背景
在经过 7 轮基于 Doubao 大语言模型自动调优超参数的实验后，模型的 mAP50 性能指标在 `0.936` 左右达到瓶颈。
根据对之前结果的分析发现，仅仅依靠数据增强策略（如 MixUp、Copy-Paste）以及超参数（如学习率、动量等）的调整，已经无法有效解决数据集中“细小病灶”和“微小特征”难以被检测到的问题。
因此，我们决定直接对网络架构进行干预，实施“Plan A + B”方案以提升模型的特征提取和微小目标检测能力。

## 2. 方案详解

### 2.1 Plan A: 引入 CBAM 注意力机制 (Convolutional Block Attention Module)
- **原理**: CBAM 包含通道注意力（Channel Attention）和空间注意力（Spatial Attention），能让网络在特征图中自适应地聚焦于重要区域，抑制无关的背景噪声。
- **改动位置**: 
  - 在 `ultralytics/nn/tasks.py` 的解析逻辑 `parse_model` 中注册了 `CBAM` 模块。
  - 在 `yolov8-gold.yaml` 中，在各个尺度的检测头特征融合输出处（P2, P3, P4, P5 层向上融合后）添加了 `CBAM` 层，使得在最终输入到 `Detect` 头之前，特征图的注意力得到进一步增强。

### 2.2 Plan B: 增加 P2 检测头 (微小目标检测层)
- **原理**: 标准 YOLOv8 默认使用 P3, P4, P5 三个尺度的特征图进行检测，其中 P3 (下采样 8 倍) 对于分辨率极小或非常密集的细小病灶特征（如鱼体表面的微小充血或囊肿）提取能力有限。引入 P2 (下采样 4 倍) 特征图，可以保留更高分辨率的细节，显著提升小目标的召回率。
- **改动位置**: 
  - 在 `yolov8-gold.yaml` 中，原本 `head` 结构最低只下探到 P3 层。本次改动中，从 P3 层通过 `nn.Upsample` 继续上采样，并与 backbone 的 P2 特征（第 2 层输出）进行 `Concat`。
  - 随后使用 `C2f` 和 `CBAM` 处理 P2 特征，并将其一路向上融合回 P3, P4, P5，形成包含 P2 到 P5 的全尺度特征金字塔。
  - `Detect` 模块的输入也从 `[P3, P4, P5]` 改为 `[P2, P3, P4, P5]`。

## 3. 预期收益
- **mAP 提升**: P2 层的引入预计将显著提升模型在微小病灶上的表现，有望突破当前 `0.936` 的瓶颈，向 `0.96` 的目标迈进。
- **鲁棒性**: CBAM 的加入将使模型对复杂背景（如鱼体反光、水质浑浊等）具有更强的抗干扰能力，从而提升 mAP50-95 的综合表现。

## 4. 下一步计划
完成源码与 YAML 的修改后，我们将重启基于 `llm_auto_tuner.py` 的大模型自动调优训练流程。新一轮训练的 Prompt 会被更新，明确告知 Doubao 模型当前架构已升级（包含 P2 头和 CBAM），让 LLM 基于全新的架构重新评估合适的超参数。
