# 4. YOLO-FD 模型调优过程与结果

## 4.1 调优思路

在完成 YOLO-FD 模型的初步部署与测试后，为进一步提升模型在实际应用中的推理效率，本阶段主要针对模型部署与推理加速进行调优尝试。具体调优思路如下：

1. **半精度（FP16）推理**：默认情况下 PyTorch 使用 FP32（单精度浮点数）进行推理。通过将模型输入及权重转换为 FP16，可以降低显存/内存占用，并在支持硬件加速的设备上（如支持 FP16 的 GPU 或特定 CPU）提升推理速度。
2. **ONNX 模型导出与加速**：ONNX（Open Neural Network Exchange）是一种针对机器学习所设计的开放式的文件格式。将 PyTorch 训练得到的 `.pt` 模型导出为 `.onnx` 格式后，可以使用专门优化过的 ONNX Runtime 进行推理，通常能实现脱离 PyTorch 庞大框架的轻量化部署，并在多平台上获得推理加速。

## 4.2 调优过程

我们编写了自动化测试脚本 `export_onnx.py` 来实现上述调优并进行性能基准测试。

### 4.2.1 依赖安装与准备
导出 ONNX 模型需要安装相应的依赖库：
```bash
pip install onnx>=1.12.0 onnxsim>=0.4.17 onnxruntime onnxscript
```

### 4.2.2 核心代码实现
在 `export_onnx.py` 中，我们实现了以下三个主要测试流程：
1. 加载原生 `yolo-fd.pt` 并进行 FP32 标准推理。
2. 开启 `half=True` 参数，测试 PyTorch FP16 推理速度。
3. 调用 `model.export(format="onnx", simplify=True)` 将模型转换为 ONNX 格式，并调用 ONNX Runtime 进行推理测试。

*注：在推理时我们指定了 `data="ultralytics/datasets/norcardis_disease.yaml"`，以覆盖模型权重内部绑定的训练设备绝对路径，避免路径解析报错。*

## 4.3 调优结果与性能对比

我们在当前环境（纯 CPU 环境）下执行了调优脚本，截取连续 10 次推理的平均时间作为参考，结果如下：

| 推理后端及配置 | 平均推理耗时 (ms/image) | 相对基准提升 |
| :--- | :--- | :--- |
| **PyTorch FP32 (基准)** | 209.26 ms | - |
| **PyTorch FP16** | 205.62 ms | 略微提升 (~1.7%) |
| **ONNX Runtime** | 291.24 ms | 负优化 (变慢) |

### 4.3.1 结果分析与结论
1. **FP16 推理**：在 CPU 环境下，FP16 推理仅有微小的性能提升（约从 209ms 降至 205ms）。这是因为当前 CPU 对 FP16 指令集的加速支持有限。如果在 GPU 设备（如 Tensor Core）上运行，FP16 的加速效果将非常显著。
2. **ONNX Runtime 推理**：在本机的纯 CPU 测试中，ONNX 推理耗时（291.24 ms）反而高于原生 PyTorch（209.26 ms）。出现这种现象的原因通常在于：
   - 当前安装的 `onnxruntime` 默认 CPU 版本的算子优化（或多线程并行策略）未完全匹配当前 CPU 架构。
   - PyTorch 在本机环境下可能已经充分调用了底层的 OpenMP/MKL 等线性代数优化库。
   - 尽管在此次 CPU 测试中 ONNX 未带来速度提升，但成功导出的 `yolo-fd.onnx` 文件极大地降低了部署的依赖门槛（无需安装体积庞大的 PyTorch），并且它是后续进行 TensorRT、OpenVINO 等特定硬件极限加速的必备前置格式。

**后续优化建议**：若要在生产环境进一步榨取性能，建议将导出的 ONNX 模型部署至支持 CUDA/TensorRT 的英伟达显卡设备，或使用 Intel OpenVINO 框架进行 CPU 端专有加速。
