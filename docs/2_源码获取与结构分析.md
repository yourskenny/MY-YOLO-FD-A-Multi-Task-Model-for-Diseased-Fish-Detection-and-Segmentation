# 2_源码获取与结构分析

## 1. 源码获取过程

### 获取方式
- **原仓库地址**: [https://github.com/feifei-Lee/YOLO-FD](https://github.com/feifei-Lee/YOLO-FD)
- **获取环境**: Windows系统（C:\coding\YOLO-FD）
- **获取方法**: 
  1. 由于测试环境中未安装Git工具，无法直接使用`git clone`。
  2. 原本尝试下载默认分支（master分支），但发现该分支中仅有一个未发布源码的 `README.md`，提示“代码将在近期发布”。
  3. 经过进一步探查发现，实际源码已被上传至 `main` 分支。
  4. 最终使用 Python 脚本，通过 GitHub 的镜像代理站（ghproxy.net）下载了 `main` 分支的 Zip 压缩包（`main.zip`），并在本地成功解压。

### 环境依赖
根据 `requirements.txt` 和官方 `README.md`，项目基础环境如下：
- **Python**: 3.8
- **CUDA**: 11.8
- **PyTorch**: 1.10
- **基础框架**: 基于 Ultralytics YOLOv8.0.117 修改

## 2. 源码结构分析

解压后的项目整体结构基于经典的 YOLOv8（Ultralytics）进行扩展和修改。以下是项目的核心目录和文件分布分析：

### 2.1 根目录核心文件
```text
C:\coding\YOLO-FD
├── train.py                # 模型训练的入口脚本，用于设置数据集和超参数
├── val.py                  # 模型验证脚本，用于在验证集上评估模型性能
├── predict.py              # 模型预测与推理脚本，用于处理图像或视频
├── requirements.txt        # 项目的Python依赖包列表
├── yolo-fd.pt              # 训练好的 YOLO-FD 模型权重
├── yolov8s.pt              # 官方 YOLOv8s 预训练权重
├── README.md               # 项目介绍、使用说明和环境要求
├── LICENSE                 # MIT 开源许可证
```

### 2.2 核心模块（`ultralytics` 文件夹）
该文件夹是算法模型的核心，包含了多任务学习（MTL）、损失函数、以及网络结构的具体修改：

1. **配置文件 (`ultralytics/models/v8/`)**
   - 新增了多种模型结构配置文件，例如 `yolov8-seg.yaml`, `yolov8-biformer.yaml`, `yolov8-swinT.yaml` 等。
   - 这表明作者在原始 YOLOv8 的基础上尝试了引入多种 Transformer 结构（如 Biformer、Swin Transformer）来增强特征提取能力。

2. **自定义网络层与模块 (`ultralytics/nn/modules/`)**
   - `Biformer.py`: 实现了 Biformer 注意力模块。
   - `Ctxt.py`: 对应论文中的 Context Aggregation 模块。
   - `GD.py`: 可能用于全局上下文或梯度相关的特征处理。
   - `SwiT.py`: Swin Transformer 模块。
   - 这些自定义层被集成到 YOLOv8 的 Backbone 或 Neck 中，用于提升模型对鱼病区域的检测和分割精度。

3. **优化与损失机制 (`ultralytics/yolo/utils/`)**
   - `pcgrad.py`: 实现了 **PCGrad**（Projecting Conflicting Gradients），用于解决多任务学习（检测和分割）中的梯度冲突问题。
   - `cagrad.py`: 实现了 CAGrad，这同样是一种多任务学习中的梯度优化方法。
   - `grad_norm.py`: 梯度标准化方法。
   - 作者引入了上述优化策略以平衡目标检测（鱼体定位）和语义分割（病灶分割）两个分支的权重和梯度更新，这与论文中所提到的“Weight uncertainty and PCGrad are employed”一致。

### 2.3 其他重要目录
- **`pre_weights/mtl/pcgrad+uncert/`**: 存放了多任务学习（MTL）结合 PCGrad 和不确定性权重策略训练得到的各种评估结果，如 `P_curve.png`, `R_curve.png`, `confusion_matrix.png` 以及训练日志和权重（`best.pt`, `last.pt`）。
- **`datasets/`**: 用于存放或配置训练所需的数据集文件。根据 `README.md` 的说明，项目使用了 Nocardiosis 鱼病数据集，但由于隐私问题并未公开。
- **`demo/`**: 用于存放测试图片和测试视频。
- **`docs/`**: 用于存放项目学习、文献阅读和源码分析相关的文档资料。

## 3. 总结
YOLO-FD 并不是一个从零编写的框架，而是建立在 **Ultralytics YOLOv8 (v8.0.117)** 基础上的二次开发。
项目的主要改动点在于：
1. **多任务学习 (MTL)**：同时实现了对象检测（检测鱼类）和语义分割（分割病灶）。
2. **多任务梯度优化**：在 `utils` 中引入了 PCGrad 和不确定性权重（Uncertainty Weighting），以解决双任务训练时带来的梯度冲突。
3. **特征增强网络**：在 `nn/modules` 中加入了 BiFormer、SwinT 等 Transformer 结构，进一步强化病灶特征的提取。
