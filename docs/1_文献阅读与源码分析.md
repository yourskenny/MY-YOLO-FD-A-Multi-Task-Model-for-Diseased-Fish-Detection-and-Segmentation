# 文献阅读与源码分析

## 1. 文献基本信息
- **中文标题**: YOLO-FD：一种基于多任务学习的精准鱼病检测
- **英文标题**: YOLO-FD: An accurate fish disease detection method based on multi-task learning
- **作者**: Xuefei Li, Shili Zhao, Chunlin Chen, Hongwu Cui, Daoliang Li, Ran Zhao
- **发表期刊/会议**: Expert Systems With Applications (2024)
- **论文链接**: [ScienceDirect](https://www.sciencedirect.com/science/article/pii/S0957417424019523)
- **开源代码仓库**: [GitHub - feifei-Lee/YOLO-FD](https://github.com/feifei-Lee/YOLO-FD)

## 2. 论文核心内容摘要
本文提出了一种专为同时进行检测（Detection）和分割（Segmentation）而设计的多任务学习网络——**YOLO-FD**。
主要贡献如下：
1. **多任务学习架构**：扩展了 YOLOv8 的主干网络（Backbone），在保留原有目标检测分支（用于识别受感染的病鱼）的基础上，集成了一个新颖的语义分割分支，专门用于精确分割病鱼的感染区域。
2. **训练优化策略**：在 YOLO-FD 的训练过程中引入了权重不确定性（Weight uncertainty）和 PCGrad 算法，以平衡和优化多任务学习的梯度。
3. **应用效果**：实现了精准的鱼类病害检测，并能够评估鱼病害的严重程度。

## 3. 数据集与实验环境
- **数据集**: 
  - VOC2012 语义分割数据集。
  - Nocardiosis (诺卡氏菌病) 鱼病数据集（该数据集由作者构建，由于某些原因未公开）。
- **实验环境**: 
  - 操作系统 / 硬件: 运行在 NVIDIA RTX 3060 显卡上
  - Python: 3.8
  - CUDA: 11.8
  - PyTorch: 1.10
  - 核心框架: Ultralytics YOLOv8.0.117

## 4. 源码运行与部署分析
开源仓库提供了基于 YOLOv8 进行修改的多任务网络代码，以下为主要的部署与运行方式：
- **依赖安装**:
  ```bash
  pip install -r requirements.txt
  ```
- **数据准备格式**:
  需要将数据集按照检测（det）和分割（seg）分为两个子目录进行存放，具体结构如下：
  ```
  📁 norcardia_disease_fish
  ├── 📁 det
  │   ├── 📁 images (train/val)
  │   ├── 📁 labels (train/val)
  ├── 📁 seg
  │   ├── 📁 labels (train/val, 以及自动生成的 class_weights.cache)
  ```
- **模型训练**:
  需要在 `ultralytics/datasets` 中配置自定义的数据集 yaml 文件，并在 `train.py` 中设置好路径，之后运行：
  ```bash
  python train.py
  ```
- **模型推理/测试**:
  仓库在 `demo` 文件夹下提供了测试图片和视频，并提供了预训练模型 `yolo-fd.pt`。
  ```bash
  python predict.py
  ```

## 5. 结论
YOLO-FD 是一项将目标检测与语义分割结合起来评估病害严重程度的优秀工作，其源码对 YOLOv8 的多任务扩展有很高的参考价值，后续可基于该框架替换或应用到我们自己的鱼病检测数据集中。
